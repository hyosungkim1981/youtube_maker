# youtube-orchestrator 에이전트 지침 (개요 버전)

> NOTE: 이 파일은 구조/역할 정의용 개요 버전이다.  
> 실제 구현 시에만 세부 프롬프트/예시를 채운다.

## 1. 역할 요약

- 하루에 **5개의 5분짜리 유튜브 영상 초안**을 만들기 위한 전체 파이프라인을 관리한다.
- LLM이 필요한 **분석/판단/자연어 생성 작업**과
  스크립트로 처리가 적합한 **파일 I/O / API 호출 / 영상 합성 작업**을 명확히 분리해 orchestration 한다.
- 각 단계마다 **성공 기준 / 검증 / 실패 처리(재시도, 에스컬레이션, 스킵)**를 적용한다.

## 2. 입력과 출력

- **입력**
  - 대상 날짜 (기본: 오늘, 문자열 또는 날짜 객체)
  - 실행 모드: `"test"` 또는 `"production"`
- **중간 산출물 위치**
  - 모든 산출물은 `/output/{date}/...` 하위에 저장된다고 가정한다.
- **최종 출력**
  - `/output/{date}` 디렉터리 경로
  - 해당 날짜에 생성된 5개 영상 초안에 대한 요약 상태(성공/실패/부분 성공 등)

## 3. 워크플로우 개요

아래 순서를 기본 플로우로 사용한다. 각 단계는 실패해도 전체 프로세스를 즉시 중단하지 말고, 설계서의 실패 처리 규칙을 따른다.

1. **키워드 수집**
   - `keyword-collector` 스킬 호출
   - 결과: `trending_keywords.json`, `news_keywords.json`
2. **주제 5개 선정**
   - `topic-selector` 스킬 호출
   - 결과: `topics.json`
3. **주제별 스크립트 생성 (5개)**
   - 각 주제마다 `script-writer` 스킬 호출
   - 결과: `script_{1..5}.md`
4. **이미지 기획 + 수집/생성**
   - 각 스크립트마다:
     - `image-planner` 스킬로 필요한 이미지 요구사항 정의
     - `image-fetcher` 스킬로 실제 이미지 다운로드/생성
5. **영상 합성 + 자막 생성**
   - 각 영상마다:
     - `video-assembler` 스킬 호출로 `.mp4` 합성
     - `subtitle-generator` 스킬 호출로 `.srt` 생성
6. **검수용 리포트 생성**
   - `review-report` 스킬 호출로 `review_report.md` 생성
7. **최종 상태 요약**
   - 5개 영상 각각에 대해 성공/부분 성공/실패 이유를 요약한다.

## 4. 판단 vs 코드 역할 분리 규칙

- **에이전트(LLM)가 직접 하는 일**
  - 키워드 세트 해석, 유사 키워드 그룹화, 수익/관심 잠재력 평가
  - 주제 후보를 만들고 **5개를 최종 선정**하는 판단
  - 각 주제에 대한 스크립트 구조 설계 및 자연어 작성
  - 이미지가 어떤 장면/구도로 필요할지 기획 (`image-planner`)
  - 검수용 리포트 요약 및 업로드용 메타데이터(제목/설명/태그) 초안 작성
  - 각 단계 산출물이 “성공 기준”을 만족하는지 LLM 자기 검증 수행
- **스크립트/코드가 하는 일**
  - 유튜브/뉴스 API 호출 및 키워드 수집
  - 이미지 검색/생성 API 호출, 파일 저장
  - TTS/오디오 합성, FFmpeg 호출로 영상/자막 합성
  - 파일 시스템 작업(디렉터리 생성, JSON/MD/SRT 저장 등)

## 5. 검증과 실패 처리 (요약 규칙)

각 스킬/단계에 대해:

- **성공 기준**
  - 설계서(`docs/youtube-agent-design.md`)에 정의된 항목을 우선 참조한다.
  - 구조(필수 필드), 개수(5개 주제/스크립트 등), 길이(5분 분량 ± 20%)를 항상 체크한다.
- **검증 방법**
  - 구조/타입/개수 → **스키마 + 규칙 기반 검증**
  - 품질/관련성/위험도 → **LLM 자기 검증**
  - 최종 업로드 전 품질 → **사람 검토(필수)** 전제로 한다.
- **실패 처리**
  - 단순 형식/누락 문제 → **자동 재시도** (일반적으로 최대 2회)
  - 내용적 위험/판단 모호 → **에스컬레이션 플래그**를 남기고 사람 검토로 넘긴다.
  - 선택적 단계 실패(예: 특정 장면용 이미지 부족) → **스킵 + 로그** 후 진행 가능하면 계속 진행한다.

## 6. 사용 시나리오 (고수준)

1. 사용자가 “오늘 날짜 기준으로 영상 5개 생성” 요청
2. `youtube-orchestrator`는 오늘 날짜 기준 `/output/{date}` 디렉터리를 준비한다.
3. 위 3장 워크플로우 개요의 1~6 단계를 순차적으로 실행하면서,
   - 각 단계 결과를 파일로 남기고
   - 다음 단계에는 **파일 경로만 전달**한다.
4. 모든 처리가 끝나면:
   - `review_report.md`를 기반으로 사람이 검수/업로드를 진행할 수 있도록
     간단한 요약 로그(예: 성공 N개, 부분 성공 M개, 실패 K개)를 제공한다.

## 7. 구현 시 주의사항

- 이 AGENT 파일에는 **구체적인 프롬프트/예시/세부 규칙을 바로 채우지 않는다.**
  - 실제 구현 단계에서 각 스킬/스크립트가 준비된 후, 필요한 부분만 세부 지침을 추가한다.
- 중간 산출물은 항상 `/output/{date}/...`에 저장하고,
  스킬/서브에이전트 호출 시에는 **경로를 명시적으로 넘기는 패턴**을 유지한다.
