# review-report 스킬 (개요 버전)

## 1. 역할

- 각 영상에 대해:
  - 주제/요약
  - 핵심 메시지/포인트
  - 민감/주의 포인트
  - 업로드 시 권장 제목/설명/태그 초안
  을 포함한 검수용 리포트를 생성한다.

## 2. 입력

- `/output/{date}/topics.json` 경로
- `/output/{date}/script_{n}.md` 경로들
- (옵션) 영상/썸네일 정보 메타데이터

## 3. 출력

- `/output/{date}/review_report.md`

예시 구조:

```markdown
# [YYYY-MM-DD] 유튜브 영상 검수 리포트

## 영상 1
- 주제: ...
- 요약: ...
- 핵심 포인트:
  - ...
- 민감/주의 포인트:
  - ...
- 권장 제목: ...
- 권장 설명: ...
- 권장 태그: ...
```

## 4. 성공 기준

- 5개 영상 각각에 대해 위 항목이 모두 존재한다.
- 운영자가 이 리포트만 보고도 업로드 준비(제목/설명/태그 작성)를 할 수 있을 정도로 정보가 충분하다.

## 5. 검증과 실패 처리

- **검증**
  - 섹션/필드 누락 여부 확인 (규칙 기반)
  - LLM 자기 검증:
    - 요약이 원 스크립트와 일치하는지,
    - 민감/주의 포인트가 적절히 포착되었는지 확인.
- **실패 처리**
  - 누락/구조 문제 → 자동 재생성(1~2회).
  - 여전히 일관성 부족 시, 해당 영상에 “사람이 반드시 수동 검토 필요” 플래그를 명시한다.
