## 유튜브 영상 자동 생성/업로드 에이전트 시스템 설계서

### 1. 작업 컨텍스트 문서

#### 1.1 배경

- 유튜브 채널 운영에서 **지속적인 콘텐츠 생산**과 **트렌드 반영**이 핵심 경쟁력이다.
- 특히 **뉴스/최신 이슈 기반 요약 콘텐츠**는 수요가 높지만, 아이디어 발굴 → 스크립트 작성 → 이미지 수집/생성 → 영상 합성 → 업로드까지의 과정이 반복적이고 시간이 많이 든다.
- 이를 자동화하여, 운영자는 **검수와 전략적 의사결정**에 집중하고, 반복 작업은 에이전트 시스템이 처리하도록 하는 것이 목표다.

#### 1.2 목적

- **매일 5개의 5분짜리 유튜브 영상 초안**(스크립트, 이미지, 영상 파일, 자막 포함)을 자동으로 생성한다.
- 운영자는 에이전트가 만든 결과물을 **검수/수정 후 직접 업로드**한다.
- 장기적으로는, 유튜브 업로드와 수익화를 통해 **안정적인 자동 콘텐츠 생산 파이프라인**을 구축하는 것이 목표다.

#### 1.3 범위

- **포함 범위**
  - 최근 1주일 유튜브 트렌드 키워드 및 최신 뉴스 키워드 수집
  - 키워드 분석 및 5개 주제 선정
  - 각 주제별 **5분 분량 스크립트 초안** 생성
  - 스크립트에 맞는 **이미지 수집/생성** (썸네일 후보 포함 가능)
  - 스크립트 + 이미지 기반 **영상(초안) 합성** 및 자막 생성
  - 중간 산출물(`scripts`, `images`, `video draft`, `subtitles`)을 파일로 저장
  - 검수용 리포트 생성 (각 영상별 요약/포인트/주의사항)
- **제외 범위**
  - 실제 **유튜브 업로드 및 메타데이터 최종 설정** (사람이 담당)
  - 수익 지표 분석, A/B 테스트, 고급 최적화 전략
  - 저작권/상표 관련 법률 자문 (단, 기본적 주의사항은 가이드로 명시 가능)

#### 1.4 입출력 정의

- **입력**
  - 최근 1주일 유튜브 영상들의 키워드 세트  
    - 예: 외부 수집 스크립트 또는 API를 통해 얻은 `trending_keywords.json`
  - 최신 뉴스 키워드 세트  
    - 예: 뉴스 API/크롤러 결과 `news_keywords.json`
  - (옵션) 채널 톤/브랜드 가이드 문서 (`/docs/channel-style-guide.md`)
- **출력**
  - 매 실행마다 **5개의 5분짜리 영상 초안** 관련 산출물:
    - 각 영상별 스크립트 (`/output/{date}/script_{n}.md`)
    - 각 영상별 이미지 목록 (파일 또는 URL, `/output/{date}/images_{n}/...`)
    - 각 영상별 영상 초안 파일 (`/output/{date}/video_{n}.mp4`)
    - 각 영상별 자막 파일 (`/output/{date}/subtitles_{n}.srt` 또는 `.vtt`)
    - 검수용 요약 리포트 (`/output/{date}/review_report.md`)
  - 하루 단위로 5개 세트 생산: **1일 5개, 성공 시 TRUE**

#### 1.5 제약조건

- **콘텐츠 길이**: 각 영상은 **약 5분 분량**을 목표로 한다.  
  - 스크립트 기준: 말하기 속도 1분당 130~160단어(한국어 환산) 등을 고려한 **총 글자수/문단 수** 가이드 적용.
- **내용 스타일**
  - 간결하고 이해하기 쉬운 구성
  - 과도한 낚시성/선정적 표현은 지양
- **생산 목표**
  - 최소 **1일 5개 영상 초안** 생성
- **사람 검수 필수**
  - 에이전트가 만든 산출물은 **운영자가 반드시 검수 후 업로드**한다.

#### 1.6 용어 정의

- **트렌드 키워드**: 최근 1주일 유튜브에서 많이 소비된 주제/검색어.
- **뉴스 키워드**: 동일 기간 내 주요 뉴스 헤드라인/기사에서 추출된 핵심 주제.
- **영상 초안(draft)**: 자동 합성된 상태의 영상으로, 색감/효과/보이스 등은 최소 구성일 수 있음.
- **검수(review)**: 사람이 스크립트/이미지/영상 내용을 검토하고, 문제 있으면 수정하거나 폐기하는 단계.

### 2. 워크플로우 정의

#### 2.1 전체 흐름 개요

1. **키워드 수집 단계** (코드 중심)
2. **키워드 분석 및 주제 5개 선정** (LLM 판단)
3. **각 주제별 5분 스크립트 생성** (LLM 생성 + 검증)
4. **이미지 수집/생성** (코드 + LLM 보조)
5. **영상 합성 및 자막 생성** (코드 중심)
6. **검수용 리포트 생성** (LLM 요약)
7. **검수 대기 상태로 마무리** (사람이 업로드)

각 단계는 **성공 기준 / 검증 방법 / 실패 처리 방식**을 명시한다.

#### 2.2 단계별 상세 플로우

##### 2.2.1 키워드 수집 단계

- **역할**
  - 최근 1주일 기준 유튜브 트렌드 키워드 수집
  - 최신 뉴스 키워드 수집
- **처리 주체**
  - **코드 처리 영역** (스킬 스크립트 + 외부 API)
- **입력**
  - 날짜 범위 (기본: 오늘 기준 -7일)
- **출력**
  - `/output/{date}/trending_keywords.json`
  - `/output/{date}/news_keywords.json`
- **성공 기준**
  - 각 JSON에 **최소 N개(예: 30개 이상)** 키워드 존재
  - 필수 필드(키워드, 출처, 빈도 등) 포함
- **검증 방법**
  - **스키마 검증**: JSON 구조 및 필드 타입 체크
  - **규칙 기반 검증**: 키워드 수 최소 개수, 중복 비율 제한
- **실패 시 처리**
  - **자동 재시도**: 동일 API 재호출 (최대 2회)
  - 여전히 실패 시 **스킵 + 로그**: 해당 날짜의 수집 실패 원인을 로그로 남기고 워크플로우 중단

##### 2.2.2 키워드 분석 및 주제 5개 선정

- **역할**
  - 유튜브 트렌드 키워드와 뉴스 키워드를 통합 분석
  - 중복·유사 키워드 클러스터링
  - **수익/관심도 잠재력이 높을 만한 5개 주제** 선정
- **처리 주체**
  - **LLM 판단 영역** (에이전트)
- **입력**
  - `trending_keywords.json`, `news_keywords.json`
- **출력**
  - `/output/{date}/topics.json`  
    - 5개 주제, 각 주제에 대해: 제목, 한줄 요약, 선택 이유, 주요 포인트 리스트
- **성공 기준**
  - 정확히 **5개 주제**가 정의되어 있음
  - 각 주제에 최소 3개 이상의 서브 포인트 존재
  - 뉴스/트렌드 어느 한쪽에만 치우치지 않고, **혼합/연계** 시도
- **검증 방법**
  - **스키마 검증**: JSON 필드 체크
  - **LLM 자기 검증**:  
    - “각 주제가 해당 키워드 세트와 충분히 관련 있는지”,  
    - “너무 중복된 주제는 아닌지”를 LLM이 자체 평가
- **실패 시 처리**
  - **자동 재시도**: 자기 검증에서 “중복/관련성 부족” 판정 시, 다른 조합으로 재생성(최대 2회)
  - 여전히 실패 시 **에스컬레이션**: 사람이 주제 목록을 확인/수정하도록 마킹

##### 2.2.3 각 주제별 5분 스크립트 생성

- **역할**
  - 선정된 5개 주제 각각에 대해, 약 5분 분량의 스크립트를 생성
- **처리 주체**
  - **LLM 생성 영역**
- **입력**
  - `topics.json`, (옵션) 채널 톤/스타일 가이드
- **출력**
  - `/output/{date}/script_{1..5}.md`
- **성공 기준**
  - 각 스크립트:
    - 도입–전개–정리 구조를 가진다.
    - 총 분량이 **5분 ± 20%** 범위 내 (글자 수/문단 수 기준 추정)
    - 과도한 선정성/허위 정보 지양 (기본 가이드 준수)
- **검증 방법**
  - **규칙 기반 검증**
    - 문단 수/글자 수 등으로 길이 추정
  - **LLM 자기 검증**
    - “정보 왜곡/선정성/혐오 표현 여부” 1차 필터링
    - “해당 주제에 대해 중요한 포인트 누락 여부” 검토
- **실패 시 처리**
  - 길이/구조 관련 실패 → **자동 재시도** (최대 2회)
  - 내용적 위험(정치/민감 이슈, 허위 가능성 등) → **에스컬레이션**: 사람이 해당 스크립트 검토/수정 또는 폐기

##### 2.2.4 이미지 수집/생성

- **역할**
  - 각 스크립트에 맞는 이미지 세트를 준비:
    - 장면 전환용 배경 이미지
    - 핵심 키워드 강조용 오버레이 이미지
    - (옵션) 썸네일 후보 이미지
- **처리 주체**
  - **코드 + LLM 보조**
  - LLM은 “어떤 타입/구도의 이미지가 필요한지”를 기획,  
    코드는 이미지 검색 API/생성 API 호출 및 파일 저장 담당.
- **입력**
  - `script_{n}.md`, `topics.json`
- **출력**
  - `/output/{date}/images_{n}/image_{k}.(png|jpg)`
  - 메타데이터: `/output/{date}/images_{n}/images_meta.json` (장면/타임라인 매핑 정보)
- **성공 기준**
  - 영상 길이에 비례하여 **최소 이미지 개수** 확보 (예: 5분 기준 10~20개)
  - 각 이미지에 대해 “어느 구간에서 사용할지” 정보 존재
- **검증 방법**
  - **스키마 검증**: `images_meta.json` 필수 필드 체크
  - **규칙 기반 검증**: 이미지 파일 존재 여부, 개수 하한
- **실패 시 처리**
  - 이미지 부족/실패 → **자동 재시도** (다른 검색어/프롬프트 사용, 최대 2회)
  - 특정 이미지 타입 반복 실패 → 해당 장면은 **단색 배경+텍스트** 등 대체 전략으로 **스킵 + 로그**

##### 2.2.5 영상 합성 및 자막 생성

- **역할**
  - 스크립트, 이미지, TTS를 조합해 5분짜리 영상 초안 파일 생성
  - 스크립트 기반 자막 파일(SRT/VTT) 생성
- **처리 주체**
  - **코드 처리 영역** (FFmpeg 등 도구 사용)
- **입력**
  - `script_{n}.md`, `images_{n}`, (선택) TTS 오디오 클립
- **출력**
  - `/output/{date}/video_{n}.mp4`
  - `/output/{date}/subtitles_{n}.srt`
- **성공 기준**
  - 영상 파일이 정상적으로 생성되고 재생 가능
  - 자막 파일과 영상 길이가 일정 범위 내에서 일치 (예: ±5초)
- **검증 방법**
  - **규칙 기반 검증**
    - 파일 존재 여부, 재생 테스트(간단 duración 체크)
    - 자막 타임스탬프 범위 검증
- **실패 시 처리**
  - 합성 실패 → **자동 재시도** (구성을 단순화한 템플릿으로 1회 재시도)
  - 여전히 실패 시 해당 영상은 **스킵 + 로그**, 다른 영상 처리 계속 진행

##### 2.2.6 검수용 리포트 생성

- **역할**
  - 운영자가 검수/업로드를 쉽게 할 수 있도록, 각 영상별 요약/주의 포인트를 정리한 리포트를 생성
- **처리 주체**
  - **LLM 요약/분석 영역**
- **입력**
  - `topics.json`, `script_{n}.md`, 간단한 메타정보(예: 예상 썸네일, 핵심 키워드)
- **출력**
  - `/output/{date}/review_report.md`  
    - 각 영상별:
      - 주제/요약
      - 핵심 메시지
      - 민감/주의 포인트
      - 업로드 시 권장 제목/설명/태그 초안
- **성공 기준**
  - 5개 영상 모두에 대해 항목이 존재
  - 운영자가 업로드 준비를 할 수 있을 정도의 정보 포함
- **검증 방법**
  - **스키마 검증**: 리포트 구조/섹션 유무 확인
  - **LLM 자기 검증**: “요약이 원 스크립트를 적절히 반영하고 있는지” 평가
- **실패 시 처리**
  - 정보 누락/구조 문제 → **자동 재시도** (1~2회)
  - LLM 스스로 일관성 부족 판단 → 재생성 또는 **사람 검토 플래그** 표시

##### 2.2.7 사람 검수 및 업로드

- **역할**
  - 에이전트가 생성한 스크립트/영상/자막/리포트를 사람이 검토 후, 실제 유튜브에 업로드
- **처리 주체**
  - **사람(운영자)** + (옵션) 보조 에이전트
- **입력**
  - `/output/{date}` 폴더 전체 산출물
- **출력**
  - 유튜브에 업로드된 실제 영상
- **성공 기준**
  - 사람이 “업로드 가능한 품질”이라고 판단한 최소 1~5개 영상 업로드
- **검증 방법**
  - **사람 검토** (필수)
- **실패 시 처리**
  - 품질 부족 시:
    - 스크립트/이미지/영상 일부만 수정하여 재활용
    - 전체 폐기 후 다음 날 주기에 재도전

### 3. 구현 스펙 (구조 개요)

#### 3.1 폴더 구조 (에이전트/스킬 기준)

```text
/project-root
  ├── CLAUDE.md                         # 메인 오케스트레이터 에이전트 지침 (레포 공용 CLAUDE와 별개로, 필요 시 추가 정의)
  ├── /.claude
  │   ├── /skills
  │   │   ├── /keyword-collector
  │   │   │   ├── SKILL.md
```

#### 3.2 CLAUDE.md 핵심 섹션 목록 (구조만)

- **프로젝트 개요**
  - 이 시스템의 목적, 범위, 주요 워크플로우 요약
- **에이전트 역할**
  - 메인 오케스트레이터(`youtube-orchestrator`)의 책임
  - 서브에이전트/스킬과의 관계
- **워크플로우 시나리오**
  - “하루 5개 영상 생산”을 위한 단계별 호출 순서
- **판단 vs 코드 분리 규칙**
  - LLM이 해야 할 결정/분석/생성 vs 스크립트가 해야 할 I/O/합성 작업 정의
- **검증/실패 처리 가이드**
  - 각 단계에서 어떤 검증 패턴 사용, 재시도/에스컬레이션 조건
- **보안/저작권 주의사항**
  - 뉴스/이미지/영상 소재 사용 시 저작권/약관 관련 기본 정책

#### 3.3 에이전트 구조

- **메인 에이전트**
  - 이름: `youtube-orchestrator`
  - 역할:
    - 일 단위 파이프라인 전체 오케스트레이션
    - 각 스킬 호출 순서/조건 관리
    - 실패/예외 상황에 대한 최종 판단 (재시도 or 종료)
- **서브에이전트 (선택)**
  - `script-review-assistant`
    - 역할: 사람이 검수할 때, 스크립트 품질 개선 제안/수정 초안 제공
    - 사용 시점: 리뷰 단계에서 운영자가 수동 호출

#### 3.4 작업 단계별 처리 방식 (LLM vs 스크립트)

- **LLM 판단/생성 영역**
  - 키워드 분석 및 주제 선정 (`topic-selector` 스킬)
  - 스크립트 생성 (`script-writer` 스킬)
  - 이미지 기획 (어떤 장면에 어떤 이미지가 필요한지, `image-planner`)
  - 검수용 리포트 요약/정리 (`review-report`)
  - (옵션) 스크립트 리뷰/개선 제안 (`script-review-assistant`)
- **스크립트/코드 처리 영역**
  - 유튜브/뉴스 키워드 수집 (`keyword-collector/scripts`)
  - 이미지 검색/생성 API 호출 (`image-fetcher/scripts`)
  - TTS 호출, 오디오 합성
  - 영상 합성, 자막 파일 생성 (`video-assembler`, `subtitle-generator`)
  - 파일 I/O, 디렉토리 생성, 날짜별 결과 저장

#### 3.5 스킬/스크립트 파일 목록, 역할, 트리거 조건

- **`keyword-collector`**
  - 역할: 트렌드/뉴스 키워드 수집
  - 트리거: 하루 1회, 워크플로우 시작 시
- **`topic-selector`**
  - 역할: 키워드 분석 → 5개 주제 선정
  - 트리거: `trending_keywords.json`, `news_keywords.json` 생성 후
- **`script-writer`**
  - 역할: 각 주제별 5분 스크립트 생성
  - 트리거: `topics.json` 생성 후, 주제당 1회
- **`image-planner`**
  - 역할: 스크립트 기반 장면/이미지 요구사항 정의
  - 트리거: `script_{n}.md` 생성 후
- **`image-fetcher`**
  - 역할: 이미지 검색/생성 및 저장
  - 트리거: `image-planner` 결과(요구사항 리스트) 생성 후
- **`video-assembler`**
  - 역할: 이미지 + 스크립트 + 오디오로 영상 합성
  - 트리거: 해당 영상의 스크립트, 이미지 세트, (옵션) TTS가 준비된 후
- **`subtitle-generator`**
  - 역할: 스크립트 → 자막 파일 변환
  - 트리거: `script_{n}.md` 준비 후
- **`review-report`**
  - 역할: 검수용 리포트 생성
  - 트리거: 모든 스크립트 및 기본 메타데이터 준비 후

#### 3.6 서브에이전트 사용 시 스펙 (예시)

- **`youtube-orchestrator` (메인)**
  - **입력**
    - 날짜(기본: 오늘), 실행 모드(테스트/실서비스)
  - **출력**
    - `/output/{date}` 산출물 및 최종 상태 요약 로그
  - **데이터 전달 방식**
    - 중간 결과는 모두 `/output/{date}/...`에 저장 후, **파일 경로**를 서브에이전트/스킬에 전달

- **`script-review-assistant` (옵션)**
  - **입력**
    - 특정 `script_{n}.md` 경로
  - **출력**
    - 개선안/수정 버전 제안 (텍스트)
  - **데이터 전달 방식**
    - 프롬프트 인라인(스크립트 일부) + 파일 경로 병행

#### 3.7 주요 산출물 파일 형식

- 키워드: `trending_keywords.json`, `news_keywords.json`
- 주제: `topics.json`
- 스크립트: `script_{n}.md`
- 이미지: `images_{n}/*.png`, `images_{n}/images_meta.json`
- 영상: `video_{n}.mp4`
- 자막: `subtitles_{n}.srt`
- 검수 리포트: `review_report.md`
